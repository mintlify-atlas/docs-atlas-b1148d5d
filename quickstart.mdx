---
title: "Run KolkataRail Intelligence: quickstart guide"
description: "Install dependencies, configure environment variables, and start all three KolkataRail services — portal, CrowdIntelligence, and NirbhayaSathi — in minutes."
---

KolkataRail Intelligence runs as three separate FastAPI services. Start them in order — NirbhayaSathi first, then Crowd Intelligence, then the portal — so the portal's proxy routes are live when you open the browser.

<Note>
  All three services must be running simultaneously. The portal on port 5000 proxies requests to port 8000 (Crowd Intelligence) and port 8001 (NirbhayaSathi). If either backend is down, the portal returns a 503 error for affected routes.
</Note>

## Prerequisites

- Python 3.11 or later
- A running Neo4j instance (required by Crowd Intelligence)
- `GROQ_API_KEY` — optional, enables AI complaint features in NirbhayaSathi
- `OPENWEATHERMAP_KEY` — optional, enables weather signal in crowd prediction

## Setup

<Steps>
  <Step title="Create a virtual environment and install packages">
    From the repository root, create a virtual environment and install all required packages:

    <CodeGroup>

    ```powershell Windows (PowerShell)
    python -m venv .venv
    .\.venv\Scripts\Activate.ps1
    pip install --upgrade pip
    pip install fastapi uvicorn[standard] httpx python-dotenv neo4j groq langdetect deep-translator
    ```

    ```bash macOS / Linux
    python3 -m venv .venv
    source .venv/bin/activate
    pip install --upgrade pip
    pip install fastapi uvicorn[standard] httpx python-dotenv neo4j groq langdetect deep-translator
    ```

    </CodeGroup>

    <Note>
      There is no `requirements.txt` in this repository. Install the packages manually using the command above.
    </Note>
  </Step>

  <Step title="Set up environment variables">
    Create a `.env` file in the repository root. At minimum, set the Neo4j credentials so Crowd Intelligence can connect:

    ```bash .env
    NEO4J_URI=bolt://localhost:7687
    NEO4J_USER=neo4j
    NEO4J_PASSWORD=your_password
    ```

    For optional AI and weather features, see the [environment variables reference](/configuration/environment-variables).
  </Step>

  <Step title="Start NirbhayaSathi (port 8001)">
    Open a terminal, navigate to the `NirbhayaSathi` directory, and start the service:

    ```powershell
    cd .\NirbhayaSathi
    python -m uvicorn main:app --host 0.0.0.0 --port 8001 --reload
    ```

    Wait for the startup log before proceeding to the next step.
  </Step>

  <Step title="Start Crowd Intelligence (port 8000)">
    Open a second terminal, navigate to the `crowdIntelligence` directory, and start the service:

    ```powershell
    cd .\crowdIntelligence
    python -m uvicorn main:app --host 0.0.0.0 --port 8000 --reload
    ```

    On startup, the service pings Neo4j. You should see `Neo4j connected` in the log. If the ping fails, check your `NEO4J_*` environment variables.
  </Step>

  <Step title="Start the portal (port 5000)">
    Open a third terminal from the repository root and start the portal:

    ```powershell
    python -m uvicorn main:app --host 0.0.0.0 --port 5000 --reload
    ```
  </Step>

  <Step title="Open the portal in your browser">
    Navigate to `http://localhost:5000/portal.html`.

    You should see the passenger-facing portal with crowd level lookup, train search, and a link to the NirbhayaSathi complaint assistant.
  </Step>
</Steps>

<Tip>
  On Windows, `start.bat` launches all three servers automatically in separate terminal windows. Update the hardcoded paths inside the file to match your local directory structure before running it.
</Tip>

## After setup

Once all three services are running, the portal gives you access to:

- **Crowd level lookup** — search a train and get a GREEN / YELLOW / RED crowd badge fused from historical, trend, live, and weather signals.
- **Train search** — find trains between any two stations, including via an intermediate stop.
- **FIR complaint filing** — start a conversational complaint session in English, Hindi, or Bengali.
- **Police dashboard** — available at `http://localhost:5000/policedashboard.html` for safety officers.
